# Linux Dev Caches Cleanup

Clear caches and build artifacts after audit classifies paths. **Ask before deleting project data or models.**

## User-level caches (generally safe)

| Path / tool | Clear command | Notes |
|-------------|---------------|-------|
| `~/.cache` (whole or subdirs) | `rm -rf ~/.cache/SUBDIR` | Spotify, browser caches re-download |
| npm | `npm cache clean --force` | ~3 GB typical |
| pnpm store | `pnpm store prune` | Check `~/.local/share/pnpm` size |
| cargo registry | `cargo cache -a` or `rm -rf ~/.cargo/registry/cache` | Keeps installed binaries in `~/.cargo/bin` |
| sccache | `sccache --stop-server && rm -rf ~/.cache/sccache` | Recompiles slower until warm |
| uv | `uv cache clean` | Python package cache |
| bun | `rm -rf ~/.cache/.bun` | Re-fetches on next run |
| pre-commit | `rm -rf ~/.cache/pre-commit` | Re-downloads hooks |
| Trash | `rm -rf ~/.local/share/Trash/*` | **Confirm** user doesn't need restores |

## Rust build artifacts

Per project:

```bash
cargo clean                    # removes target/ in cwd
```

Large `target/` dirs are the #1 Code folder consumer. Safe if you can rebuild; expect long recompile.

Find totals:

```bash
du -ch ~/Code/**/target 2>/dev/null | tail -1
```

## Node / frontend artifacts

```bash
# per project
rm -rf node_modules .next .turbo dist

# monorepo — use project scripts when available
pnpm store prune
```

## Python / devenv

| Path | Action |
|------|--------|
| `.venv/` | Delete if recreatable via `uv sync` / `pip install` / devenv |
| `.devenv/` | Delete; regenerated by `devenv shell` |
| `__pycache__/`, `.mypy_cache/` | Safe to delete |
| `data/`, `runs/` in research repos | **Not cache** — user decision |

## Nix / devenv on NixOS

```bash
nix-collect-garbage --dry-run   # preview
nix-collect-garbage -d          # delete unreachable store paths
nix-store --optimise            # hardlink duplicates
```

Never `rm` paths inside `/nix/store`.

## Git bloat

Large `.git/objects/pack`:

```bash
git gc --aggressive --prune=now
git repack -a -d -f --depth=250 --window=250
```

For 20 GB packs, consider `git filter-repo` if history contains committed binaries (separate task).

## ML / local models

```bash
du -sh ~/.lmstudio/models
# Delete specific model files via LM Studio UI or rm unused blobs after confirming names
```

~30 GB is typical for several local LLMs.

## Compiler / IDE caches

| Path | Safe? |
|------|-------|
| `~/.cache/cursor-compile-cache` | Yes |
| `~/.cache/typescript` | Yes |
| `~/.cache/nix` | Yes (download cache, not store) |
| `~/.rustup/toolchains` (old) | Remove unused: `rustup toolchain uninstall NAME` |

## Journal (system, low impact here but standard)

```bash
sudo journalctl --disk-usage
sudo journalctl --vacuum-time=7d
# or --vacuum-size=500M
```

## Priority order (typical dev workstation)

1. Docker build cache + unused images (see `linux/docker-cleanup`)
2. Rust `target/` on inactive or rebuildable projects
3. `~/.cache/sccache`, Spotify cache, browser caches
4. Trash, npm/cargo/uv caches
5. LM Studio models user no longer needs
6. Research `data/` and application datasets — only with explicit approval

## Anti-patterns

- `rm -rf ~/Code/**/target` without listing projects first
- Deleting `data/` directories assuming they are cache
- Clearing `~/.cargo/bin` when removing cargo cache
